# Privacy

Cowlick is local-first and has no analytics. Prompt and result previews are disabled by default. Full approval operations are held only long enough to show or copy a pending request.

## Stored data

`~/Library/Application Support/Cowlick/` contains the owner-only bridge token, runtime metadata, installed helper, sanitized local logs when present, `active-sessions.json`, and `provider-accounts.json`. The recovery ledger stores only the current working sessions' Codex session ID, turn ID, working directory, model when supplied, and last lifecycle timestamp. It is written atomically with owner-only permissions, prunes entries older than 24 hours, and never contains prompts, commands, results, approvals, tokens, or transcripts. Recovered entries are labeled unconfirmed and do not count as active until a fresh local lifecycle observation or trusted hook arrives. It is current lifecycle state, not a session-history database.

`provider-accounts.json` stores a version, user-chosen alias, provider identifier, account UUID, and opaque Keychain reference for each OpenAI API or Anthropic API organization-billing account. It is an owner-only file and contains no administrator credential. Credentials are device-only generic-password items in macOS Keychain. Removing an account removes its Keychain item and its metadata. Cowlick keeps each account separate and does not persist billing snapshots or history.

`~/.codex/hooks.json` contains six merged command handlers, with a timestamped backup before changes. `~/.local/bin/cowlick-hook` is a symlink to the installed helper. Preferences use `com.henryvn27.Cowlick`.

On a one-time upgrade from the development name NotchRelay, Cowlick reads only known preference keys, replaces only exact or marked legacy hook handlers, and removes the legacy helper after the new integration is installed successfully. Unrelated Codex configuration is preserved.

The app makes Sparkle update checks to the GitHub release feed. Public update archives and appcasts are accepted only when their signatures verify; if no signed feed is published, Cowlick cannot install an unsigned update. Links open only when the user selects them. There is no cloud account, advertising, or crash-reporting SDK.

Official Codex quota is read from the Codex app-server process already installed on the Mac. Cowlick requests only rate-limit data, does not read Codex's account file, and does not save quota history. It reflects the single subscription identity active in the local Codex executable Cowlick selects; Cowlick does not import, switch, or aggregate Codex subscription identities.

While Cowlick is running, its local activity observer watches `~/.codex/sessions` using FSEvents. At startup it considers recently modified JSONL files, reads the session metadata record and a bounded tail, then processes newly appended bytes. It extracts only session ID, turn ID, working directory, model, timestamp, parent-agent relationship, agent role, and the `task_started`, `task_complete`, and `turn_aborted` lifecycle markers. Raw records are held only in transient buffers; prompt, message, command, tool-input, and result fields are not extracted, logged, displayed, or persisted. This observer does not read `~/.codex/archived_sessions`, make network requests, or participate in approval decisions.

Cowlick can label a live session with the short chat name shown by the local Codex desktop catalog. “Show Codex chat names” is enabled by default and can be disabled independently. Cowlick opens the current user's `~/.codex/sqlite/codex-dev.db` read-only, matches only the exact session UUID, accepts only a local-host catalog row, sanitizes and bounds the result, and keeps it in memory only. When “Show prompt previews” is also enabled, Cowlick may fall back to the exact session's `threads.title` in `~/.codex/state_5.sqlite`; that value is often derived from the first prompt and is therefore covered by the prompt-preview privacy choice. Chat names never enter hooks, bridge messages, approvals, logs, diagnostics, preferences, or the recovery ledger. Disabling chat names clears every in-memory copy and restores project names as the primary label.

When “Estimate local Codex usage at API rates” is enabled, Cowlick scans `~/.codex/sessions` and `~/.codex/archived_sessions`. It streams bounded JSONL records and retains only allowlisted rollout identity, model, turn ID, timestamp, and numeric token-counter fields needed to deduplicate and price local usage. Cowlick also opens `~/.codex/logs_2.sqlite` read-only and incrementally reads bounded `response.create` trace rows to recognize only exact `service_tier: priority` turns. Raw trace bodies, prompts, messages, commands, and tool content are neither retained nor logged. Sanitized file summaries and matched turn IDs are cached in memory only. If Priority metadata cannot be verified, Cowlick uses Standard pricing and marks coverage partial. The estimate makes no network request, covers this Mac rather than an account, excludes unsupported models and tool fees, and is not a subscription charge or actual bill.

Opening Settings → Accounts refreshes every saved billing account; Refresh All and each account's Refresh action do the same on demand. For each account, Cowlick sends that account's Admin API key and a month-to-date interval to its provider's official endpoint: `https://api.openai.com/v1/organization/costs` for OpenAI API or `https://api.anthropic.com/v1/organizations/cost_report` for Anthropic API. OpenAI organization costs are treated as account-wide. Anthropic's official cost report excludes Priority Tier usage, so Cowlick marks Anthropic coverage as partial without applying that limitation to OpenAI. The provider also receives normal HTTPS metadata such as the user's IP address and Cowlick user-agent. Results stay in memory and remain separate per account. No prompt, approval operation, other provider credential, or Codex subscription data is sent. Cowlick does not aggregate these charges or present them as subscription usage. Actual provider billing remains separate from the local API-price equivalent.

The optional “Will Codex Reset?” forecast is disabled by default. When enabled, Cowlick makes an HTTPS request to `https://www.willcodexquotareset.com/api/forecast`. Opening the menu can refresh data older than 30 seconds; other automatic refresh triggers use a 15-minute freshness interval, and a manual refresh is immediate. There is no polling timer. The site receives the normal information inherent in a web request, such as the user's IP address and Cowlick user-agent. Cowlick decodes only the forecast score, reset-announced flag, and refresh timestamps; it does not persist the response or history. This is third-party data shown as provided. It is not Cowlick data or a Cowlick estimate, and Cowlick does not warrant it.

Core behavior needs no special privacy permission. Optional Caps Lock signaling may require Input Monitoring or Accessibility depending on macOS and hardware; the app asks only when that independent feature is enabled.

Reset App State clears in-memory state, including billing results and errors, and preferences. It reloads provider account metadata and reconciles account selection; provider accounts and Keychain credentials remain until individually removed. A normal contributor uninstall also preserves provider accounts and credentials. The explicit contributor `--purge` path deletes and verifies every referenced Keychain item before account metadata is removed, and stops with the metadata intact if verification fails. Hook removal affects only Cowlick and recognized legacy handlers. When a public Homebrew cask is available, its uninstall removes the app but does not silently delete Codex hooks or Keychain credentials; remove those explicitly in Cowlick before uninstalling. Complete source-install removal is documented in the contributor uninstall script.
